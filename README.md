# kamiwayland

C++23 Wayland client library built on C++ modules. Talks directly to the compositor socket — no libwayland dependency of
any kind. Protocol bindings are C++ module units generated from Wayland XML by the bundled `kscanner` tool.

## No libwayland dependency

kamiwayland reimplements the Wayland wire protocol directly over a Unix socket — libwayland is not linked, wrapped, or
included. There is no `libwayland-client.so` to find at runtime and no `<wayland-client.h>` required at compile time.
C++ bindings that wrap libwayland still pull hundreds of C macros, `wl_proxy`, `wl_interface`, and `wl_array` into your
translation unit even when hidden behind a nicer API.

With kamiwayland, `import wl;` exports exactly what the module declares:

```cpp
import wl;
// In scope: wl::Compositor, wl::Surface, wl::ShmFormat::argb8888, ...
// Not in scope: wl_proxy, wl_interface, WL_SHM_FORMAT_ARGB8888, wl_array, ...
```

The POSIX socket headers kamiwayland needs internally are confined to the global module fragment and never exported.

## Requirements

- GCC 15+ or Clang 18+ (Clang requires libc++) — `import std;` is used throughout
- CMake 4.2+

## Build setup

`import kamiwayland;` gives you the core runtime — the connection, wire protocol, and object system. It knows nothing
about specific Wayland interfaces like `wl_compositor` or `xdg_toplevel` — those are described in XML protocol files
maintained by the Wayland project. kscanner reads those XML files and generates one C++ module per protocol that builds
on top of kamiwayland and exposes typed classes for every interface, request, and event.

So a typical application imports three things:

```cpp
import kamiwayland;  // core runtime
import wl;           // wl::Compositor, wl::Surface, wl::Shm, ...
import xdg;          // xdg::WmBase, xdg::Surface, xdg::Toplevel, ...
```

The `wl` and `xdg` modules are generated by kscanner from `wayland.xml` and `xdg-shell.xml`.

### CMake

`kscanner_add_protocols` builds kscanner and runs it at build time to generate the protocol modules, then compiles them
into a CMake target with kamiwayland already wired in. Pass it a target name, then pairs of `<module-name> <xml-path>`:

```cmake
add_subdirectory(kamiwayland)

kscanner_add_protocols(my_protos
        wl /usr/share/wayland/wayland.xml
        xdg /usr/share/wayland-protocols/stable/xdg-shell/xdg-shell.xml
)
target_link_libraries(myapp PRIVATE my_protos)
set_target_properties(myapp PROPERTIES CXX_MODULE_STD ON)
```

`CXX_MODULE_STD ON` is required on any target that uses `import std;` directly.

### Without CMake

Run kscanner manually to generate the protocol modules:

```sh
kscanner --name wl  /usr/share/wayland/wayland.xml \
         --name xdg /usr/share/wayland-protocols/stable/xdg-shell/xdg-shell.xml \
         -o generated/
# produces: generated/wl.cppm  generated/xdg.cppm
```

Compile `src/*.cppm` as C++23 module interface units to produce the `kamiwayland` module. The generated `.cppm` files
`import kamiwayland;` so they require it — compile them the same way after. Then compile your application importing all
of them. `--name <ns>` sets the module name and C++ namespace; without it the name comes from the
`<protocol name="...">` attribute in the XML.

## Usage

**Connect:**

```cpp
auto result = kamiwayland::Display::connect();
if (!result) { std::print(std::cerr, "error: {}\n", result.error()); return 1; }
auto& display = *result;
```

**Enumerate globals:**

```cpp
auto registry = wl::get_registry(display);

std::unique_ptr<wl::Compositor> compositor;
std::unique_ptr<wl::Shm>        shm;

registry->on_global = [&](wl::Registry& reg, uint32_t name, std::string_view iface, uint32_t ver) {
    if (reg.try_bind<wl::Compositor>(iface, name, ver, compositor)) return;
    if (reg.try_bind<wl::Shm>(iface, name, ver, shm)) return;
};
if (auto r = display.roundtrip(); !r) { /* handle */ }
```

`try_bind<T>` compares `iface` against `T::interface_name`, caps the bound version at the minimum of the
compositor-advertised and client-maximum versions, writes the result into `target`, and returns true if the name
matched.

**Event callbacks:**

All events are `std::function` members named `on_<event>`. Set them before the roundtrip that delivers the events:

```cpp
seat->on_capabilities = [&](wl::Seat& s, wl::SeatCapability caps) {
    if (wl::has(caps, wl::SeatCapability::keyboard))
        keyboard = s.get_keyboard();
};
if (auto r = display.roundtrip(); !r) { /* handle */ }
```

File descriptors (keymap fd, shm fd) arrive as plain `int` parameters in the relevant callbacks, extracted from
ancillary socket data.

**Event loop:**

```cpp
struct pollfd pfd{display.fd(), POLLIN, 0};
while (running) {
    while (display.has_buffered()) {
        if (auto r = display.dispatch_once(); !r) { /* handle */ }
    }
    if (poll(&pfd, 1, -1) < 0 || !(pfd.revents & POLLIN)) break;
    if (auto r = display.dispatch_once(); !r) { /* handle */ }
}
```

A single `recvmsg` can deliver multiple messages. The inner drain loop is necessary — without it you will sleep in
`poll` while messages sit unprocessed in the buffer.

## vs libwayland

### Binding globals

**libwayland:**

```c
static struct wl_compositor *compositor;
static struct wl_seat *seat;

static void registry_global(void *data,          // userdata void* — cast back yourself
                             struct wl_registry *reg,
                             uint32_t name, const char *iface, uint32_t ver)
{
    if (strcmp(iface, wl_compositor_interface.name) == 0)  // manual string match
        compositor = (struct wl_compositor *)              // void* — wrong cast compiles, crashes
                     wl_registry_bind(reg, name, &wl_compositor_interface, 1);  // version hardcoded
    else if (strcmp(iface, wl_seat_interface.name) == 0)
        seat = (struct wl_seat *)
               wl_registry_bind(reg, name, &wl_seat_interface, 1);
}

// separate static struct required for every listener
static const struct wl_registry_listener registry_listener = {
    .global = registry_global,
};

wl_registry_add_listener(registry, &registry_listener, NULL);  // last arg: void* userdata
wl_display_roundtrip(display);
```

**kamiwayland** — `try_bind<T>` matches on `T::interface_name`, writes the result into a `unique_ptr<T>` out
parameter, caps the bound version at the minimum of the compositor-advertised and client-maximum versions, and takes
no userdata:

```cpp
std::unique_ptr<wl::Compositor> compositor;
std::unique_ptr<wl::Seat>       seat;

registry->on_global = [&](wl::Registry& reg, uint32_t name,
                           std::string_view iface, uint32_t ver) {
    if (reg.try_bind<wl::Compositor>(iface, name, ver, compositor)) return;
    if (reg.try_bind<wl::Seat>(iface, name, ver, seat)) return;
};
if (auto r = display.roundtrip(); !r) { /* r.error() */ }
```

### Event callbacks

**libwayland:**

```c
static void seat_capabilities(void *data, struct wl_seat *seat, uint32_t caps)
{
    struct app *app = data;                       // manual cast, every time
    if (caps & WL_SEAT_CAPABILITY_KEYBOARD)       // caps is a raw uint32, not a typed enum
        app->keyboard = wl_seat_get_keyboard(seat);
}

// separate static struct required for every listener
static const struct wl_seat_listener seat_listener = {
    .capabilities = seat_capabilities,
};

wl_seat_add_listener(seat, &seat_listener, app);  // app is the void* userdata
```

**kamiwayland** — assign only the callbacks you need, capture state in the lambda directly:

```cpp
seat->on_capabilities = [&](wl::Seat& s, wl::SeatCapability caps) {
    if (wl::has(caps, wl::SeatCapability::keyboard))
        keyboard = s.get_keyboard();
};
```

### Enums and type safety

All libwayland protocol enums are plain `uint32_t`. Many share the same numeric value across different interfaces, so
passing the wrong one is a silent logic error:

```c
// WL_OUTPUT_TRANSFORM_NORMAL, WL_SHM_FORMAT_ARGB8888, and WL_KEYBOARD_KEY_STATE_RELEASED
// are all 0. This compiles fine:
wl_surface_set_buffer_transform(surf, WL_SHM_FORMAT_ARGB8888);
```

kscanner generates a scoped `enum class : uint32_t` per `<enum>` in the XML. Passing the wrong type is a compile error:

```cpp
keyboard->on_key = [](wl::Keyboard&, uint32_t serial, uint32_t time,
                       uint32_t key, wl::KeyboardKeyState state) {
    if (state == wl::KeyboardKeyState::pressed) { ... }
};
```

Enums marked `bitfield="true"` get `operator|`, `operator&`, `operator~`, and `has()` generated automatically:

```cpp
if (wl::has(caps, wl::SeatCapability::keyboard)) { ... }
if (wl::has(mode, wl::OutputMode::current))       { ... }
```

### Error handling

Every fallible call returns `std::expected<T, std::string>` rather than surfacing errors through a global display error
state:

```cpp
auto result = kamiwayland::Display::connect();
if (!result) { std::print(std::cerr, "{}\n", result.error()); return 1; }

if (auto r = display.roundtrip(); !r) { /* r.error() */ }
```

### RAII and version safety

Each object's destructor sends the protocol destroy request automatically. `try_bind<T>` caps the bound version at
the minimum of the compositor-advertised and client-maximum versions. Requests with a `since` attribute check the
negotiated version at the call site and abort with a diagnostic rather than sending an invalid request the compositor
will reject with a fatal error.

### Out of scope

- Server-side code. Client only.
- `wl_cursor` — cursor theme loading and animated cursors.
- EGL / dmabuf — no `wl_egl_window`, no hardware buffer helpers.
- Event loop — `display.fd()` and `dispatch_once()` are exposed; integration with poll, epoll, or an async runtime is
  left to the caller.
